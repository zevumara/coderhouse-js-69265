SPA: Single-Page Application

Las single-page application (aplicaciónes de una sola página) son sitios web donde todo el contenido pasa por un solo HTML (index.html) pero a medida que vamos ingresando a diferente secciones se va actualizando dinámicamente utilizando el DOM.

También pueden cargar contenido externo (APIs) o de otros archivos (.json) asíncronicamente sin necesidad de recargar la página.

Por ejemplo, las aplicaciones hechas con React son aplicaciones SPA.

✍️ Características

SPA: Single-Page Application

Estas son algunas características de las SPA:

  1. Estructura de página única: En una SPA, toda la aplicación se carga en una sola página web al principio y luego se actualiza dinámicamente según las interacciones del usuario, utilizando JavaScript. No se requiere cargar páginas adicionales desde el servidor para la mayoría de las acciones.
  2. Interfaz de usuario dinámica: Las SPAs proporcionan una experiencia de usuario fluida y rápida, ya que los cambios de contenido ocurren sin necesidad de recargar toda la página.
  3. Mejor para aplicaciones interactivas: Las SPAs son ideales para aplicaciones web que requieren una gran cantidad de interacción del usuario y actualizaciones en tiempo real, como redes sociales, aplicaciones de correo electrónico y aplicaciones de chat.
  4. Uso extensivo de AJAX: Las SPAs suelen utilizar solicitudes AJAX para obtener y enviar datos al servidor sin necesidad de recargar la página. Esto contribuye a una experiencia de usuario más rápida y eficiente.
  5. Mayor carga inicial: Las SPAs a menudo tienen una carga inicial más pesada, ya que se deben descargar todos los recursos necesarios (HTML, JavaScript, CSS) al principio. Sin embargo, las interacciones posteriores son más rápidas.
